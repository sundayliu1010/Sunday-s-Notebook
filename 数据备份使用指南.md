# 💾 数据备份与恢复使用指南

## 🌸 功能概述

Hello Kitty Daily Notebook 现已支持完整的数据备份和恢复功能，帮助您安全地保存和迁移您的笔记数据。

### ✨ 主要功能
- **📤 数据导出**：将所有笔记、待办事项、设置等数据保存为备份文件
- **📥 数据导入**：从备份文件恢复您的所有数据
- **📊 数据统计**：实时显示笔记数量、待办事项统计、数据大小等信息
- **🔒 本地安全**：备份文件仅保存在您的设备上，确保数据隐私

---

## 🚀 快速开始

### 访问设置页面
1. 点击顶部导航栏的 **⚙️ 设置** 按钮
2. 进入 **"🌸 Hello Kitty 设置"** 页面
3. 找到 **"💾 数据备份与恢复"** 区域

### 查看数据统计
在备份区域，您可以实时看到：
- 📝 **笔记数量**：当前保存的笔记总数
- 📋 **待办事项**：未完成的任务数量
- ✅ **已完成**：已完成的任务数量
- ⚙️ **API Key**：显示是否已配置OpenAI API Key
- 💾 **数据大小**：估算的本地数据存储大小

---

## 📤 导出数据

### 导出步骤
1. **检查数据状态**：确认有可导出的数据（导出按钮可用）
2. **点击导出**：点击 **"📤 导出数据"** 按钮
3. **等待处理**：系统会收集所有数据并生成备份文件
4. **自动下载**：备份文件会自动下载到您的默认下载文件夹

### 备份文件信息
- **文件格式**：JSON格式 (.json)
- **文件命名**：`Hello Kitty Daily Notebook_YYYY-MM-DD.json`
- **包含内容**：
  - 所有笔记内容
  - 待办事项列表
  - 用户设置信息
  - OpenAI API Key（已配置的话）
  - 应用主题设置

### 导出示例
```
文件名：Hello Kitty Daily Notebook_2024-01-15.json
大小：约 50KB
内容：您的所有笔记、待办事项和设置
```

---

## 📥 导入数据

### 导入步骤
1. **点击导入**：点击 **"📥 导入数据"** 按钮
2. **选择文件**：在弹出的文件选择器中选择备份文件（.json格式）
3. **文件验证**：系统会自动验证文件格式和大小
4. **确认导入**：查看导入预览信息，确认后点击确定
5. **等待完成**：系统会导入数据并自动刷新页面

### 导入预览信息
```
即将导入以下数据：
📝 笔记: 15 篇
📋 待办事项: 8 个
⚙️ API Key: 已配置

这将覆盖当前所有本地数据。确定要继续吗？
```

### 文件要求
- **格式**：必须是 .json 格式的备份文件
- **大小**：不超过 10MB
- **来源**：建议使用本应用导出的备份文件

## 🔧 高级功能

### 数据验证机制
系统会自动验证备份文件的：
- ✅ 文件格式正确性
- ✅ 数据结构完整性
- ✅ 必要字段存在性
- ✅ 数据类型匹配性

### 错误处理
如果遇到问题，系统会提供明确的错误提示：
- 🚫 **文件格式错误**：请选择有效的备份文件
- 🚫 **文件过大**：请选择小于10MB的文件
- 🚫 **解析失败**：备份文件可能已损坏

### 安全保护
- 🔒 **本地处理**：所有数据处理都在您的浏览器本地完成
- 🔒 **无服务器传输**：数据不会上传到任何外部服务器
- 🔒 **隐私保护**：API Key等敏感信息仅保存在本地

## 💡 使用建议

### 定期备份
建议您：
- 📅 **每周备份**：养成定期备份的习惯
- 🎯 **重要节点**：完成重要笔记后及时备份
- 🔄 **多处保存**：将备份文件保存在多个安全位置

### 备份时机
在以下情况下建议备份：
- ✨ 写完重要的日记或笔记后
- ✨ 完成阶段性的待办事项后
- ✨ 更换了设备或浏览器后
- ✨ 重装系统前

### 文件管理
- 🏷️ **重命名建议**：可以按日期重命名备份文件便于管理
- 💾 **存储位置**：建议保存在云盘或外部存储设备
- 🗑️ **清理旧文件**：定期清理过期的备份文件

## 🛠 故障排除

### 常见问题

**Q: 导出按钮不可用？**
A: 可能是因为当前没有可导出的数据。请先创建一些笔记或待办事项。

**Q: 导入失败？**
A: 请检查：
- 备份文件是否完整
- 文件格式是否为 .json
- 文件大小是否超过 10MB
- 文件是否来自本应用的导出

**Q: 导入后数据不完整？**
A: 可能的原因：
- 备份文件本身数据不完整
- 导入过程中网络中断
- 浏览器存储空间不足

**Q: 忘记备份了怎么办？**
A: 很抱歉，如果您的本地数据已经被清除且没有备份，我们无法恢复数据。请养成定期备份的习惯。
